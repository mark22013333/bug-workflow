# feature-update — 更新功能開發進度

在功能開發過程中，將需求變更、DB 調整、架構調整、開發進度等資訊，即時寫入 Notion 功能頁面的對應區塊。

---

## 設定檔

執行前依序檢查以下路徑，讀取第一個找到的設定檔：
1. `~/.claude-company/feature-workflow-config.md`（公司環境）
2. `~/.claude/feature-workflow-config.md`（個人環境）

若都不存在，提示使用者先執行 `/feature-setup`。

---

## 前置條件

- 已使用 `/feature-start` 建立功能條目

---

## 流程

### 1. 定位目標功能 Notion 頁面

搜尋 Notion「任務追蹤工具」中符合條件的條目：

使用 `notion-search` 搜尋：
- 狀態為「進行中」
- 任務類型包含「💬 功能要求」

同時取得 Git Repo 識別碼（從 `git remote get-url origin` 解析）和 Git branch，用於輔助篩選。

優先匹配邏輯：
1. 若「修復分支」欄位與當前 Git branch 完全匹配 → 自動選定
2. 若只有 1 筆進行中的功能需求（同一專案下）→ 自動選定
3. 若有多筆候選，優先顯示與當前 Git Repo 所屬專案相關的條目
4. 若有多筆候選 → 列出清單讓使用者選擇
5. 若無候選 → 提示使用者先用 `/feature-start` 建立

選定後，使用 `notion-fetch` 取得頁面完整內容，以便後續 `update_content` 操作。

### 2. 判斷更新類型

根據使用者輸入的內容，自動判斷應更新哪個區塊：

| 使用者輸入特徵 | 目標區塊 | 說明 |
|--------------|---------|------|
| 包含「需求變更」、「需求調整」、「新增需求」 | 📋 需求描述 | 附加需求變更記錄 |
| 包含「DB 調整」、「加欄位」、「改表」、「新增欄位」 | 🗄️ 資料庫設計 | 附加 DB 變更記錄 |
| 包含「架構調整」、「架構變更」、「改分層」 | 🏗️ 架構設計 | 附加架構變更記錄 |
| 包含「API 調整」、「新增 API」、「改 API」 | 📐 技術規格 | 附加 API 變更記錄 |
| 包含「測試」、「驗證」、「通過」、「失敗」 | 🧪 測試計畫 | 更新測試狀態 |
| 使用者明確指定區塊 | 指定區塊 | 依使用者指示 |
| 其他（一般進度、筆記、決策） | 📝 開發日誌 | 附加開發日誌 |

若無法自動判斷，預設寫入「📝 開發日誌」。

### 3. 更新 Notion 頁面

使用 `notion-update-page` 的 `update_content` 指令：

**原則**：
- **附加而非覆蓋**：同一區塊可多次更新，每次新內容附加在既有內容之後
- **加上時間戳**：每次更新前加上 `[{日期} {HH:mm}]` 時間標記
- **格式化**：程式碼用 code block、SQL 用 sql code block、一般文字用 bullet point

### 4. 可選更新開發階段

若使用者的更新暗示階段變更，主動詢問：

```
偵測到此更新可能涉及階段變更，是否更新開發階段？
當前階段：規格設計
1. 維持不變
2. 更新為：DB 設計
3. 其他（手動選擇）
```

### 5. 回傳結果

向使用者回傳：
- 已更新的區塊名稱
- 更新內容的前 2 行預覽
- Notion 頁面連結

## 快捷用法彙整

```
/feature-update 需求變更：客戶要求新增匯出 Excel 功能   → 更新需求描述
/feature-update DB 調整：msg_main 新增 export_status 欄位 → 更新資料庫設計
/feature-update 架構調整：改用 Strategy 模式處理匯出格式   → 更新架構設計
/feature-update API 調整：新增 GET /api/export/{id}       → 更新技術規格
/feature-update 完成 Service 層實作，進入 Controller 開發  → 更新開發日誌
/feature-update 單元測試全部通過                           → 更新測試計畫
```

## 邊界情況

- **設定檔不存在**：提示使用者先執行 `/feature-setup` 完成初始設定
- **頁面內容與標準模板不符**：嘗試模糊匹配區塊標題，找不到則附加在頁面最後
- **更新內容過長（> 200 行）**：自動截斷，保留前 50 行和後 20 行，中間以 `... (省略 N 行) ...` 替代
- **多次更新同一區塊**：新內容附加在既有內容之後，以時間戳區隔
